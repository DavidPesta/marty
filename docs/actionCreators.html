Action Creators

When new data arrives into your application it needs to be processed by the stores before being passed to the views (Called a data flow). Action creators are responsible for formatting that data in a standardised way so that it can be processed by the stores. It does this by creating an action which is composed of a type, some data and (optionally) a source (e.g. from the view, server).

At their most basic, an action creator is a function that calls this.dispatch with an action type (Ideally from a constant) and any data you want to pass to the store.

var UserActionCreators = Marty.createActionCreators({
  name: 'Users',
  createUser: function (name, email) {
    this.dispatch(Constants.Users.ADD_USER, name, email);
  }
});

var UserStore = Marty.createStore({
  handlers: {
    createUser: Constants.Users.ADD_USER
  },
  createUser: function (name, email) {
    this.state.push({
      name: name,
      email: email
    });
    this.hasChanged();
  }
})

Internally the dispatch function will create a new action which is passed to the dispatcher which it to Store#handleAction.

var action = {
  source: 'VIEW',
  type: 'ADD_USER',
  arguments: ['foo', 'foo@bar.com']
}

The second responsiblity of action creators is to coordinate optimistic actions and server responses. For example, if you were to create a new user you will likely want to add it to a local store and then start synchronising with the server in the hopes that the request will successfully complete. If for whatever reasons that request fails you need a way to easily rollback any changes to the stores.

The dispatch function actually returns the action after all stores have finishing processing it. The action has a rollback() function on it will ask the stores to rollback any changes they've made (if possible).

var UserActionCreators = Marty.createActionCreators({
  createUser: function (user) {
    var action = this.dispatch(Constants.Users.ADD_USER, user);

    // if you wanted to rollback straight away
    action.rollback();

    // or you could pass it to the API
    UserAPI.saveUser(user, action.rollback);
  }
});

API

createActionCreators(props)

To create some new action creators, you call Marty.createActionCreators passing in a set of properties. It returns your action creators as a singleton.

var UserActionCreators = Marty.createActionCreators({
  createUser: function (name, email) {
    this.dispatch(Constants.Users.ADD_USER, name, email);
  }
});

name

An (optional) display name for the action creator. Used by Marty Developer Tools.

dispatch(actionType, [...])

Creates a new action, with the action type being the first argument. The remaining arguments will be the arguments for any store action handlers.

The actions source will be null.

Returns Action. You can rollback an action by calling action.rollback().

dispatchViewAction(actionType, [...])

Creates a new action, with the action type being the first argument. The remaining arguments will be the arguments for any store action handlers.

The actions source will be VIEW (or Marty.constants.actionSources.VIEW).

Returns Action. You can rollback an action by calling action.rollback().

dispatchServerAction(actionType, [...])

Creates a new action, with the action type being the first argument. The remaining arguments will be the arguments for any store action handlers.

The actions source will be SERVER (or Marty.constants.actionSources.SERVER).

Returns Action. You can rollback an action by calling action.rollback().

Server Action Creators

Action creators often will call an Http API which, when complete, will then call another action creator with the new data. If you are using a module loader (e.g. CommonJS, AMD) it can cause a cyclic dependency between the Action Creator and the HTTP API.

The way to get around this is to have a seperate action creator, called a Server Action Creator, that is responsible for handling responses from Http APIs.

var UserActionCreators = Marty.createActionCreators({
  saveUser: function (user) {
    UserAPI.createUser(user);
  }
});

var UserServerActionCreators = Marty.createActionCreators({
  receiveUser: function (user) {
    this.dispatch(Constants.Users.ADD_USER, user);
  }
});

var UserAPI = Marty.createHttpAPI({
  createUser: function (user) {
    this.post({ url: '/users', data: user }).then(function (user) {
      UserServerActionCreators.receiveUser(user);
    });
  }
});

Actions vs Action Creators

There has been some confusion around actions and action creators. In the original blog announcing Flux, action creators were called actions. Since then, the creators of Flux have decided to rename actions to action creators because "they create actions but are not themselves actually the action itself".

Utilmately this is just semantics so actions and action creators are terms that can be used interchangeably.
